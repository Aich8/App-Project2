# Documentation

This folder contains project documentation outside the application code.

## Structure

- `specs/business/`: business requirements and product boundaries.
- `specs/global.md`: high-level product purpose, audience, and behavior.
- `specs/functional/`: feature behavior and user flows.
- `specs/technical/`: implementation and storage rules.
- `specs/functional-map.md`: index of functional spec files.
- `plans/active/`: current implementation plans.

Application code lives in `../website/`.
